## 🚨 Проблема: ERR_CONNECTION_REFUSED

### Почему возникает?

```
Vercel (облако)  ──X──>  localhost:8080 (ваш компьютер)
     ❌ НЕТ ДОСТУПА
```

**Причины:**
1. **localhost** = адрес локального компьютера
2. Vercel работает на серверах в **другой сети**
3. Ваш компьютер **не доступен из интернета**
4. Firewall блокирует входящие соединения

### Аналогия

Представьте:
- Ваш компьютер = **ваша квартира**
- localhost:8080 = **ваша комната**
- Vercel = **офис в другом городе**

Офис не может зайти в вашу комнату, потому что:
- Нет адреса квартиры (публичного IP)
- Дверь закрыта (firewall)
- Нет ключа (нет маршрута)

## ✅ Решения

### Решение 1: Деплой бэкенда в облако (Рекомендуется)

**Railway / Render / Fly.io**

```
Vercel (облако)  ──✅──>  Railway (облако)
     ✅ ПРЯМОЙ ДОСТУП
```

**Преимущества:**
- ✅ Стабильное соединение
- ✅ HTTPS из коробки
- ✅ Автоматический деплой
- ✅ Масштабирование
- ✅ Логи и мониторинг

**См. `ДЕПЛОЙ_BACKEND.md` для инструкций**

### Решение 2: Ngrok (Временное)

**Только для разработки!**

```
Vercel (облако)  ──✅──>  Ngrok (туннель)  ──✅──>  localhost:8080
     ✅ РАБОТАЕТ, НО...
```

**Недостатки:**
- ❌ URL меняется при перезапуске
- ❌ Требует постоянно включенный компьютер
- ❌ Лимиты на бесплатном tier
- ❌ Не подходит для production

### Решение 3: VPS с публичным IP

**Аренда сервера (DigitalOcean, Hetzner)**

```
Vercel (облако)  ──✅──>  VPS (публичный IP)
     ✅ ПОЛНЫЙ КОНТРОЛЬ
```

**Преимущества:**
- ✅ Полный контроль
- ✅ Статический IP
- ✅ Можно настроить любые сервисы

**Недостатки:**
- ❌ Требует администрирования
- ❌ Нужно настраивать HTTPS
- ❌ Дороже Railway

## 🔐 CORS: Как это работает

### Что такое CORS?

**Cross-Origin Resource Sharing** - механизм безопасности браузера.

**Проблема:**
```
https://your-app.vercel.app  ──X──>  https://backend.railway.app
         (origin 1)                        (origin 2)
     ❌ РАЗНЫЕ ДОМЕНЫ
```

Браузер блокирует запросы между разными доменами **по умолчанию**.

### Наше решение

**Backend (server.go):**
```go
// Разрешаем запросы от Vercel
if origin != "" {
    if len(origin) > 11 && origin[len(origin)-11:] == ".vercel.app" {
        allowedOrigins[origin] = true
    }
}

w.Header().Set("Access-Control-Allow-Origin", origin)
w.Header().Set("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE, OPTIONS")
w.Header().Set("Access-Control-Allow-Headers", "Content-Type, Authorization")
```

**Разрешенные origins:**
- `http://localhost:3000` - Next.js dev
- `http://localhost:5173` - Vite dev
- `*.vercel.app` - Production

### Preflight запросы

Браузер сначала отправляет **OPTIONS** запрос:

```
OPTIONS /api/v1/generate
Origin: https://your-app.vercel.app
Access-Control-Request-Method: POST
```

Backend отвечает:

```
Access-Control-Allow-Origin: https://your-app.vercel.app
Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS
Access-Control-Max-Age: 3600
```

Только после этого браузер отправляет **реальный POST** запрос.

## 🔧 Переменные окружения

### Frontend (Vercel)

```bash
# .env.production
VITE_API_BASE_URL=https://istok-backend.railway.app/api/v1
VITE_USE_MOCKS=false
```

**Как добавить в Vercel:**
1. Откройте проект в Vercel Dashboard
2. Settings → Environment Variables
3. Добавьте `VITE_API_BASE_URL`
4. Выберите "Production"
5. Redeploy

### Backend (Railway)

```bash
PORT=8080
OPENROUTER_API_KEY=sk-or-v1-xxxxx
GO_ENV=production
```

**Как добавить в Railway:**
1. Откройте проект в Railway Dashboard
2. Variables
3. Добавьте переменные
4. Railway автоматически перезапустит сервис
