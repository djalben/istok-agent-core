### 🔬 ФАЗА 2: Модернизация Backend

#### Reasoning Engine (Размышления агента)
**Файл:** `internal/domain/reasoning.go`

Агент теперь **анализирует задачу перед выполнением**:
1. **Анализ** - оценка сложности и требований
2. **Декомпозиция** - разбиение на подзадачи
3. **Планирование** - выбор оптимальной стратегии
4. **Валидация** - проверка корректности подхода
5. **Оптимизация** - улучшение решения

```go
// Пример использования
reasoningEngine := domain.NewReasoningEngine(agent)
chain, err := reasoningEngine.Reason(ctx, task)
// Получаем детальное размышление с уверенностью и заключением
```

#### UI Extractor (Извлечение UI-паттернов)
**Файл:** `internal/infrastructure/crawler/ui_extractor.go`

WebCrawler теперь извлекает:
- **UI Компоненты** - Button, Card, Modal, Navigation, Form, DataGrid
- **Цветовую палитру** - доминирующие цвета сайта
- **Типографику** - используемые шрифты
- **Layout паттерны** - Grid, Flexbox, Sidebar, Hero

```go

```

## 🔄 Полная цепочка работы

1. **Пользователь** вводит URL конкурента или выбирает шаблон
2. **WebCrawler** парсит сайт, извлекает UI-паттерны и дизайн-систему
3. **Reasoning Engine** анализирует задачу (5 шагов размышления)
4. **Agent Intelligence** оценивает риски и выбирает стратегию
5. **Code Generator** генерирует код с использованием:
   - Накопленных знаний (Learning Context)
   - Извлеченных паттернов
   - Мульти-модельной архитектуры (fallback)
6. **Frontend** отображает результат в реальном времени

## 🎯 Следующие шаги

### Для полной интеграции istok-agent UI:

1. **Переключить на реальный backend**
   ```typescript
   // istok-agent/src/lib/api.ts
   const USE_MOCKS = false;
   ```

2. **Запустить istok-agent**
   ```bash
   cd istok-agent
   bun run dev
   ```

3. **Или интегрировать компоненты в /web**
   - Скопировать компоненты из `istok-agent/src/web/components`
   - Адаптировать под Next.js структуру
   - Использовать существующий API клиент

### Для активации Reasoning:

Раскомментировать в `generate_project.go`:
```go
// ФАЗА РАЗМЫШЛЕНИЯ
reasoningChain, err := s.reasoningService.ReasonAboutTask(ctx, task)
```

### Для Media Generation:

Добавить API ключи:
```bash
REPLICATE_API_KEY=...
GOOGLE_AI_KEY=...
```
